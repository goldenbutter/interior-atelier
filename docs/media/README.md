# Media — README screenshots & GIFs

Drop the following files here and they'll render in the root [README.md](../../README.md) preview section:

| File | What it should show | Format |
|---|---|---|
| `classic-preview.gif` | A short loop walking through the classic homepage (hero → gallery → footer) | GIF, ≤ 8 MB, ≤ 1280px wide |
| `premium-preview.gif` | A short loop highlighting the premium scroll-explode hero, material library, testimonials | GIF, ≤ 8 MB, ≤ 1280px wide |
| `classic-home.png` | Static screenshot of the classic homepage at desktop width | PNG, ≤ 1MB, 1440×900 sweet spot |
| `premium-home.png` | Static screenshot of the premium homepage at desktop width | PNG, ≤ 1MB, 1440×900 sweet spot |
| `classic-gallery.png` | Static screenshot of the classic Elvebooking-style gallery section | PNG |
| `premium-materials.png` | Static screenshot of the premium MaterialLibrary bento grid | PNG |

Tips:
- Capture against the real production URLs so the GIFs reflect what's actually shipped (`https://demo-lysning-{classic,premium}.ibithun.com`).
- For GIFs, [LICEcap](https://www.cockos.com/licecap/) or [ScreenToGif](https://www.screentogif.com/) work well on Windows. Export at 12–15 fps and crop to the relevant section to keep file size down.
- For static screenshots, Chrome DevTools → Cmd-Shift-P → "Capture full size screenshot" gives clean shots without browser chrome.
